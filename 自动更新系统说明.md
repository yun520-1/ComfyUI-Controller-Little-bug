## 🚀 使用方法

### 方法 1: 手动执行一次

```bash
cd ~/.jvs/.openclaw/workspace/ComfyUI-Controller-Little-bug
python3 github_auto_updater.py --once
```

### 方法 2: 只扫描不更新

```bash
python3 github_auto_updater.py --scan-only
```

### 方法 3: 启动定时任务

```bash
# 安装定时任务
./安装定时任务.sh

# 查看定时任务
crontab -l

# 删除定时任务
crontab -l | grep -v run_updater.sh | crontab -
```

### 方法 4: 后台运行

```bash
# 使用 nohup
nohup python3 github_auto_updater.py --interval 2 > updater.log 2>&1 &

# 查看进程
ps aux | grep github_auto_updater

# 停止进程
pkill -f github_auto_updater
```

## ⚙️ 配置选项

### 更新周期配置

在 `github_auto_updater.py` 中修改：

```python
GITHUB_REPO = "yun520-1/ComfyUI-Controller-Little-bug"  # 仓库
OPTIMIZATION_COUNT = 3  # 优化次数要求
```

### 定时任务配置

编辑 `安装定时任务.sh`：

```bash
CRON_EXPR="0 */2 * * *"  # 每 2 小时
```

常用 cron 表达式：
- `0 */2 * * *` - 每 2 小时
- `0 * * * *` - 每小时
- `0 0 * * *` - 每天午夜
- `0 0 */2 * *` - 每 2 天

## 📊 更新流程

```
开始
  ↓
扫描 GitHub (检查提交/相关项目)
  ↓
发现更新？
  ├─ 否 → 结束
  └─ 是 ↓
优化代码 (第 1 轮)
  ↓
优化代码 (第 2 轮)
  ↓
优化代码 (第 3 轮)
  ↓
验证代码 (语法/导入/测试)
  ↓
验证通过？
  ├─ 否 → 取消发布，记录日志
  └─ 是 ↓
准备 Release (版本号/CHANGELOG)
  ↓
推送到 GitHub
  ↓
创建 Release
  ↓
保存日志
  ↓
结束
```

## 🛡️ 安全保障

### 验证机制
1. **语法验证**: 确保所有 Python 文件语法正确
2. **导入验证**: 确保模块可以正常导入
3. **执行验证**: 运行测试确保功能正常
4. **优化次数**: 至少优化 3 次才发布

### 错误处理
- 验证失败 → 取消发布
- 推送失败 → 记录日志
- 网络错误 → 等待下次执行
- 语法错误 → 自动跳过

### 回滚机制
```bash
# 查看 Git 历史
git log --oneline

# 回滚到上一个版本
git reset --hard HEAD~1
git push --force
```
